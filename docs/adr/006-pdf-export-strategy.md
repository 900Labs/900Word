# ADR 006: PDF Export Strategy

## Status

Accepted

## Decision

The product provides two PDF/print paths:

- A WebView print flow that renders sanitized print HTML with page setup CSS.
- A lightweight generated PDF writer for deterministic text-oriented export.

The generated PDF writer paginates a bounded lightweight body projection using `word-core` page setup, emits one page object per generated page, honors explicit page breaks, repeats simple header/footer text, resolves page-number/page-count/date fields deterministically, and supports validated page ranges.

The body projection supports text lines, simple table rows/cells, and simple figure placeholders. Tables render as vector cell boxes with wrapped cell text. Image blocks render as visible placeholder boxes with safe visible alt/caption text and rough alignment/scale handling from `ImageBlock` presentation metadata. The PDF text stream does not emit image asset IDs, original source names, local paths, usernames, or hostnames as image metadata.

Sprint 037 adds bounded active PDF URI annotations for safe external `http`, `https`, and `mailto` inline text links in paragraphs, headings, list text, and table cell text. Link rectangles are approximate text-run areas produced by the lightweight layout pass, and annotation objects are capped per page and per exported document. Safe internal `#bookmark` links remain visible text only in generated PDFs.

Raster image embedding, merged cells, table resizing, rich table styling, formulas, complex nested table layout, embedded/subset fonts, internal PDF destinations, exact glyph-level link geometry, complex script shaping, page-bottom footnote layout, and editor-preview/full print-layout fidelity remain deferred until a layout-engine ADR is accepted.

## Consequences

- README must not claim full desktop-publishing PDF fidelity.
- PDF tests cover header/xref/EOF smoke, multi-page object structure, explicit page breaks, header/footer page fields, page-range validation, table border drawing and cell text, image placeholder text/privacy, structured-block pagination, safe external URI annotations, annotation bounds, page-range behavior with annotations, and absence of private path/build metadata.
- Print HTML is generated by Rust from `word-core` and must share the sanitized HTML export boundary.
- The Tauri capability file grants only the narrow WebView print permission in addition to core defaults.
