# ADR 006: PDF Export Strategy

## Status

Accepted

## Decision

The product provides two PDF/print paths:

- A WebView print flow that renders sanitized print HTML with page setup CSS.
- A lightweight generated PDF writer for deterministic text-oriented export.

The generated PDF writer paginates the existing lightweight text projection using `word-core` page setup, emits one page object per generated page, honors explicit page breaks, repeats simple header/footer text, resolves page-number/page-count/date fields deterministically, and supports validated page ranges.

Raster image embedding, embedded/subset fonts, active PDF link annotations, complex script shaping, page-bottom footnote layout, and editor-preview/full print-layout fidelity remain deferred until a layout-engine ADR is accepted.

## Consequences

- README must not claim full desktop-publishing PDF fidelity.
- PDF tests cover header/xref/EOF smoke, multi-page object structure, explicit page breaks, header/footer page fields, page-range validation, and absence of private path/build metadata.
- Print HTML is generated by Rust from `word-core` and must share the sanitized HTML export boundary.
- The Tauri capability file grants only the narrow WebView print permission in addition to core defaults.
