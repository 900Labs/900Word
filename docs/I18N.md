# Internationalization

900Word uses structured translation files for UI strings and keeps document content separate from UI locale.

Initial language work:

- English source strings in `apps/desktop/src/lib/i18n.ts`.
- Initial Spanish UI translations for the main desktop shell.
- RTL layout smoke path through the `ar` UI locale direction.
- Locale-aware dates only where user-visible.
- Translation contribution guidance before accepting community translations.

Do not hard-code new user-facing strings in Rust or Svelte components without adding them to the translation surface. Brand names, file format names, license identifiers, and keyboard event names may remain literal.

## Dictionary Locale Versus UI Locale

Spell-check dictionaries and UI translations are separate settings:

- `language_tag` selects a local UTF-8 Hunspell-shaped word-list dictionary.
- `ui_locale` selects the Svelte UI translation surface.

Missing spell-check dictionaries fall back to the bundled `en-US` bootstrap dictionary with a warning. Missing UI translations fall back per string to English. Full Hunspell affix expansion is not implemented in Sprint 006 and remains a documented follow-up.
