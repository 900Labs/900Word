# File Formats

## Native Format

OpenDocument Text (`.odt`) is the native saved format.

## Bootstrap Support

- `.odt`: MVP read/write boundary for generated documents, covering paragraphs, headings, inline marks, safe links, safe paragraph/heading bookmarks, internal fragment links, generated table-of-contents blocks, footnote/endnote references and bodies authored by 900Word, lists, tables, 900Word-authored table cell presentation metadata, page breaks, metadata title, named paragraph styles, 900Word-authored paragraph style properties, basic page setup, 900Word-authored direct paragraph/text formatting, 900Word-authored headers/footers/page fields, allowlisted embedded image bytes, locally imported JPEG bytes with accepted pre-scan APP/COM marker segments stripped, 900Word-authored image presentation metadata, 900Word-authored comments, and 900Word-authored text-only tracked changes.
- `.docx`: conversion-only import/export boundary. DOCX is not the canonical saved format. Import supports paragraphs, Heading 1-3, bold/italic/underline, safe hyperlinks, simple list fallback, simple tables, simple default/first-page headers/footers with page-number, page-count, and date fields, safe embedded PNG/JPEG/GIF/WebP image relationships, simple anchored comments, simple text-only insertion/deletion revisions, and simple footnotes/endnotes. Export writes a minimal valid package for paragraphs, headings, basic inline marks, safe hyperlinks, simple lists, simple tables, 900Word-authored page regions, valid 900Word-authored embedded image assets, simple 900Word-authored anchored comments, simple 900Word-authored text-only insertion/deletion revisions, and simple 900Word-authored local footnotes/endnotes.
- `.txt`: export plain document text to a user-entered `.txt` path.
- `.html`: export offline sanitized semantic HTML to a user-entered `.html` path.
- `.pdf`: export a valid lightweight paginated PDF byte stream to a user-entered `.pdf` path for simple sharing, with optional page-range export, simple vector table boxes, supported table-cell fill/alignment/border projection, bounded JPEG image XObjects for safe in-document JPEG assets, visible figure placeholders for unsupported image payloads, and bounded active URI annotations for safe external text links.

## Current ODT Limits

- ODT is the native saved package format, but full layout fidelity is not claimed.
- Sprint 005 page setup controls serialize and import basic page width, page height, and margin metadata. Full layout fidelity and deterministic pagination are not claimed.
- Sprint 007 print HTML uses page setup metadata for browser/WebView print margins. Full layout fidelity remains deferred.
- Sprint 032 PDF export is a lightweight deterministic text pagination adapter with generated page objects, a generated xref table, page setup sizing/margins, explicit page-break handling, repeated simple header/footer text, and page-range export. Sprint 036 extends the same bounded projection with simple table row/cell boxes, wrapped cell text, and visible figure placeholder boxes for image alt/caption text. Sprint 037 adds bounded active PDF URI annotations for safe external `http`, `https`, and `mailto` inline text links in paragraphs, headings, list text, and table cell text. Sprint 038 embeds bounded safe in-document JPEG assets as `/DCTDecode` image XObjects while keeping unsupported image payloads on the placeholder path. Sprint 048 projects supported light cell fills, hidden borders, and simple left/center/right cell text positioning; justify falls back to wrapped left-positioned text. It is not a full layout engine.
- Sprint 009 preserves 900Word-authored paragraph alignment, line spacing, paragraph spacing, indents, inline font family, font size, text color, highlight color, and list levels through generated automatic ODT style names. This is not yet a claim of full external ODT style compatibility.
- Sprint 010 preserves 900Word-authored update-style-from-selection paragraph style properties through generated ODT paragraph styles. This is limited to paragraph formatting properties and is not a full external ODT style editor.
- Sprint 011 exposes desktop hyperlink authoring for the safe external link subset. Sprint 016 extends text links to safe internal fragments. Links are limited to `http`, `https`, `mailto`, and `#bookmark-id` values whose fragment is a compact generated-safe bookmark ID. Sprint 037 PDF export emits active URI annotations only for the safe external subset.
- Sprint 012 exposes desktop editing for table cells that contain paragraphs, headings, or lists. Sprint 017 adds bounded 900Word-authored table insertion plus row/column add/delete and delete-table controls for rectangular editable tables. Sprint 048 adds bounded 900Word-authored cell presentation metadata for no/light fill choices, inherit/left/center/right/justify text alignment, and border visible/hidden state. Sprint 049 adds plain-text TSV paste into supported editable tables on the existing local paste path, bounded to the same 1-8 row/column limits and falling back for irregular input. ODT save/reopen preserves these table structures and cell presentation values with existing table handling and `word900` table-cell attributes; external table styles are not claimed as lossless. Merged cells, resizing, formulas, heavy table styles, rich HTML/spreadsheet paste, arbitrary colors, per-side borders, and unsupported nested cell content remain deferred.
- Sprint 013 stores simple section-level header/footer regions in `word-core`, persists 900Word-authored page regions through ODT `styles.xml`, and supports page-number, page-count, and date fields. External header/footer structures beyond the simple paragraph/field subset import with warnings and read-only state instead of silent rewrite. Sprint 032 PDF export renders these fields with deterministic generated page numbers, total page count, and document modified date values.
- Sprint 014 adds local image insertion for PNG, JPEG, GIF, and WebP files. Imported images are copied into `document.assets` and saved as embedded ODT package entries; source paths and source filenames are not stored. Sprint 050 strips accepted local JPEG/JPG APP0-APP15 and COM marker segments before embedding while preserving the accepted image structure and scan data. PNG, GIF, and WebP imports are unchanged by Sprint 050.
- Sprint 015 adds editable image alt text, caption text, alignment, and scale percentage. Sprint 018 adds a direct selected-image resize handle that updates the same bounded scale percentage. ODT save/reopen preserves 900Word-authored values with bounded `word900` metadata attributes on image frames. Broader ODT-native image sizing/layout compatibility is not claimed.
- Sprint 016 adds optional bookmark IDs to paragraph and heading blocks. ODT save/reopen preserves safe values with native `text:bookmark` elements inside `text:p` and `text:h`, and preserves internal links as safe `#fragment` text links. Unsafe imported bookmark names and unsafe internal hrefs are stripped with warnings. The editor exposes stable link targets only for blocks that already have safe bookmark IDs.
- Sprint 020 adds Draft/Page Layout view modes, Fit Width/100%/custom zoom controls, and simple visual rulers in the desktop editor viewport. These settings are local component state only, are not written to `.odt`, and do not add deterministic pagination, page-break preview, font metrics, or print layout fidelity claims.
- Sprint 022 adds bounded local comment threads anchored to selected text through inline comment IDs. ODT save/reopen uses ODF `office:annotation` and `office:annotation-end` elements for 900Word-authored comments, with `word900` metadata for local comment ID and resolved state. Unsafe or invalid external annotations are ignored with generic warnings. Sprint 035 adds simple DOCX comments conversion for anchored text ranges. Full external ODT annotation compatibility, replies, multi-author identity, threaded DOCX comments, DOCX resolved-state fidelity, and PDF comment annotation export are not claimed.
- Sprint 023 adds text-only tracked changes for 900Word-authored insertions and selected-text deletions. ODT save/reopen preserves local change ID, kind, author, timestamp, and recording state with `word900` metadata on inline spans. This keeps deleted text in the package until accepted or rejected. Sprint 039 adds simple DOCX insertion/deletion conversion for supported text runs. Formatting-only changes, table structure changes, image changes, move changes, resolved-state fidelity, compare/merge, multi-author collaboration, full Word review fidelity, and full external ODT change-tracking fidelity are not claimed.
- Sprint 024 adds 900Word-authored generated table-of-contents blocks. Entries are derived from supported top-level Heading 1-3 blocks in the editable first-section projection and store heading text, level, and a safe internal bookmark target. ODT save/reopen preserves the TOC through `word900:block-type="table-of-contents"` metadata on a visible generated paragraph. External ODT applications may treat it as ordinary text; full ODT-native TOC interoperability, automatic external refresh, deterministic page numbers, and page-number fidelity are not claimed.
- Sprint 025 adds bounded local footnotes and endnotes. ODT save/reopen writes 900Word-authored notes with ODF `text:note`, `text:note-citation`, and `text:note-body` elements plus bounded `word900` note ID/kind metadata. Bounded ODT notes with matching safe `word900` metadata import as local notes and are surfaced in the desktop Notes sidebar. Notes with missing, mismatched, unsafe, duplicate, over-limit, invalid, or unanchored structure import with generic warnings and visible fallback text instead of trusted hidden metadata. Sprint 040 extends this local note model to simple DOCX note conversion only. Full external ODT note interoperability, cross-reference management, note continuation, deterministic page-bottom placement, and pagination fidelity are not claimed.
- Sprint 026 smart typing is editor typed-input behavior only. It does not add ODT metadata, silently rewrite imported documents, or change import/export compatibility claims. Any transformed text is saved as ordinary document text after the user types it.
- Sprint 027 expanded statistics are desktop UI projections only. The stats panel reads existing document state, current page setup, and selection snapshots to show counts, estimated pages, estimated reading time, and local document indicators. It does not add ODT metadata, import/export behavior, page-break data, deterministic pagination, filenames, paths, usernames, hostnames, or source image filenames.
- Sprint 030 document inspection is a desktop UI projection only. The inspector reports `.odt` as the canonical saved format plus created/modified metadata timestamps, aggregate local document indicators, and privacy warnings. It does not add ODT metadata, import/export behavior, package cleanup, page-break data, deterministic pagination, filenames, paths, usernames, hostnames, recovery locations, or source image filenames.
- Sprint 014 and Sprint 015 HTML and print HTML export include allowlisted in-document image assets as `data:image/...;base64,...` URLs under the offline CSP and render image alt/caption/alignment/scale. Sprint 050 means accepted local JPEG/JPG imports are already stored without pre-scan APP/COM marker payloads before those assets reach ODT, HTML, print HTML, DOCX, or PDF export paths. Sprint 038 PDF export embeds only safe in-document baseline `image/jpeg` assets that pass bounded JPEG structure checks and metadata-marker stripping, with a cap of 32 embedded images per generated PDF, 8 MiB per embedded JPEG, 8192 px per side, 20,000,000 pixels, and grayscale/RGB component counts. PNG, GIF, WebP, malformed JPEG, oversized JPEG, unsupported component-count, post-scan metadata-marker, and over-cap cases continue to render as visible figure placeholders with alt/caption text. PDF export does not emit asset IDs, source names, local paths, usernames, hostnames, creation-date metadata, producer metadata, APP/COM JPEG metadata segments, or rich image metadata.
- Sprint 034 DOCX conversion imports safe embedded image relationships into generated `docx-image-<n>.<ext>` assets and block-level `ImageBlock`s. Mixed text/image DOCX paragraphs may become adjacent text and image blocks so visible text is not silently dropped. Image sizing, cropping, compression/downsampling, and broader DOCX media layout fidelity are not claimed.
- Sprint 035 DOCX conversion imports simple comments from safe local `word/comments.xml` or `word/comments*.xml` package parts into generated local comment IDs and inline text anchors. Point-only, unanchored, malformed, duplicate, missing, unsupported, reply/threaded, or over-limit comments degrade with generic warnings and are not stored as hidden metadata.
- Sprint 039 DOCX conversion imports simple `w:ins` and `w:del` wrappers in body paragraphs, list items, and table cells when they contain supported text runs. Imported revisions receive generated local `chg-docx-change-<n>` IDs, safe bounded author metadata, and parsed RFC3339 timestamps when present; unsafe or missing author/date metadata falls back to generic local-safe values. Nested, malformed, over-limit, move, formatting-only, property, table-row, image, field, hyperlink-specific fidelity, comment-as-change, or otherwise unsupported revision markup degrades with generic warnings and visible text fallback where text is available.
- Sprint 016 HTML and print HTML export emit safe bookmark IDs as element `id` attributes and preserve safe internal `#fragment` hrefs. Sprint 037 PDF link handling emits active annotations only for safe external URI links; active internal `#bookmark` PDF destinations remain deferred.
- Sprint 022 TXT, HTML, print HTML, and PDF export do not claim comment fidelity. Comment text, sidebars, and active annotations are not exported as PDF comments in this MVP. Sprint 035 DOCX export writes only simple 900Word-authored anchored comments; replies, threaded comments, and resolved-state fidelity are not exported.
- Sprint 023 TXT, HTML, print HTML, and PDF export do not claim track-changes fidelity. Pending inserted/deleted text may appear as normal document text in simple exports; active PDF review metadata is not emitted in this MVP. Sprint 039 DOCX export emits only simple 900Word-authored text-only insertion/deletion revision markup for supported inline text runs; external hyperlinks and comment anchors on the same inline are conservatively not exported as active DOCX link/comment metadata.
- Sprint 024 TXT and PDF export render TOCs as ordinary visible text. HTML and print HTML render TOCs as local `<nav>` blocks with safe internal fragment links where targets are safe. No export path emits or claims deterministic TOC page numbers.
- Sprint 025 TXT and PDF export render note references inline and append Footnotes/Endnotes sections as ordinary text. HTML and print HTML render local note references and append sanitized note body sections. No export path emits active PDF note annotations, deterministic page-bottom placement, or note layout fidelity claims.
- Unsupported ODT elements import with warnings.
- Unsupported or unsafe image references import with warnings instead of remote loading.
- Unsupported image payload types are rejected instead of embedded.
- Unsafe text link schemes and unsafe internal fragments are stripped during import.
## Current DOCX Limits

- ODT remains the canonical saved package format. Opening a `.docx` imports it as an unsaved dirty document and does not adopt the `.docx` path for Save.
- DOCX import reads only bounded WordprocessingML package parts: `word/document.xml`, `word/_rels/document.xml.rels`, `word/numbering.xml`, safe relationship-resolved `word/header*.xml` or `word/footer*.xml` page-region parts, safe relationship-resolved `word/comments.xml` or `word/comments*.xml` comments parts, safe relationship-resolved `word/footnotes.xml` and `word/endnotes.xml` note parts, and safe embedded image parts under `word/media/`.
- DOCX import warns generically when unsupported content is ignored or degraded. It does not surface source local paths or private filenames in warnings.
- Simple paragraphs, Heading 1-3 styles, bold, italic, underline, safe `http`, `https`, `mailto`, and safe internal-fragment hyperlinks import into `word-core`.
- Numbered and bulleted DOCX paragraphs import as generic 900Word list blocks when numbering metadata is simple enough. Unsupported numbering details are reduced to generic markers.
- Simple DOCX tables import as `word-core` tables. Nested tables are flattened to visible text with a generic warning.
- Simple default and first-page DOCX headers/footers import into `word-core` `PageRegions` when their relationship targets resolve safely under `word/` and their content is simple paragraphs/runs. `PAGE`, `NUMPAGES`, and `DATE` simple fields map to 900Word page fields where recognized. Even-page regions, complex fields, complex section layouts, and unsupported region content degrade with generic warnings.
- Safe local DOCX image relationships under `word/media/` import into `word-core` assets only when the extension/media type and magic bytes agree for PNG, JPEG/JPG, GIF, or WebP. Linked, remote, missing, oversized, unsafe, or unsupported media degrade with generic warnings.
- Simple DOCX comments import only when the comments relationship resolves safely to `word/comments.xml` or `word/comments*.xml`, the comment body/author pass `word-core` validation, and body text ranges are anchored by supported `commentRangeStart`/`commentRangeEnd` markers around visible text. Comment relationship target names, raw unsafe IDs, source filenames, paths, and unsupported hidden metadata are not preserved.
- Simple DOCX tracked insertions/deletions import only from `w:ins` and `w:del` wrappers around supported visible text runs in body paragraphs, list items, and table cells. Raw DOCX revision IDs, source filenames, paths, account metadata, usernames, hostnames, and unsupported hidden metadata are not preserved.
- Simple DOCX footnotes/endnotes import only when the standard note relationship resolves safely to `word/footnotes.xml` or `word/endnotes.xml`, the note body is made of supported paragraphs/runs with visible text, tabs, and line breaks, and body references appear in supported body paragraphs, list items, or table cells. Imported note IDs are generated locally as safe `note-docx-footnote-<n>` or `note-docx-endnote-<n>` values; raw DOCX note IDs, source filenames, paths, account metadata, usernames, hostnames, and unsupported hidden metadata are not preserved.
- DOCX move changes, formatting-only changes, property changes, table row changes, hyperlink-specific revision fidelity, comment-as-change semantics, replies, threaded comments, resolved-state fidelity, rich note formatting/layout, comments or tracked changes inside note bodies, cross-reference fidelity, custom XML, embedded objects, macros, rich styles, layout settings, merged cells, formulas, linked/remote images, broad media fidelity, and exact pagination are not imported as structured features in this MVP.
- DOCX export writes a minimal valid `.docx` package with document, styles, numbering, relationship, simple page-region parts, valid bounded embedded image media parts, a simple generated comments part when anchored 900Word comments are present, simple generated `w:ins`/`w:del` markup for 900Word-authored text-only tracked insertions/deletions, and generated `word/footnotes.xml` / `word/endnotes.xml` note parts when supported local note references are present. It does not export ODT-native metadata fidelity, replies, threaded comments, resolved-state fidelity, formatting/table/image/move changes, rich note formatting, page-bottom note placement, continuation separator fidelity, TOC fields, active PDF-like annotations, deterministic layout, image cropping/sizing fidelity, compression/downsampling, complex section layout, even-page regions, linked/remote images, compare/merge metadata, or full office-suite style fidelity.
- Binary `.doc` compatibility remains deferred.

## Deferred Support

- Broad `.docx`: comment replies/threading/resolved-state fidelity, formatting/table/image/move tracked changes, full Word review fidelity, compare/merge, rich note formatting/layout, page-bottom note placement, continuation separators, note cross references, linked/remote images, image sizing/cropping/layout fidelity, image compression/downsampling, broad media fidelity, even-page headers/footers, complex fields, complex section layout, advanced numbering, styles, layout fidelity, merged cells, embedded objects, and richer compatibility remain deferred.
- PNG/GIF/WebP PDF image embedding, progressive JPEG embedding, JPEG decoding, downsampling/recompression, crop/rotation, EXIF interpretation/selective preservation, broad malformed-JPEG metadata recovery, color-management precision, rich PDF image metadata, merged cells, table resizing, rich table styling, formulas, complex nested table layout, embedded/subset fonts, internal PDF destinations, exact glyph-level link geometry, complex script shaping, page-bottom footnote layout, and editor-preview layout fidelity remain deferred.
- `.doc`: deferred until external converter security is documented.
- `.epub`: deferred until PDF and HTML export are stable.
